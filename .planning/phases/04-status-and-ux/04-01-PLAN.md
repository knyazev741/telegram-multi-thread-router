Create `src/bot/status.py` with a `StatusUpdater` class that manages one editable status message per session.

Constructor takes: `bot: Bot`, `chat_id: int`, `thread_id: int`.

Internal state:
- `_message_id: int | None` — the Telegram message to edit
- `_current_tool: str | None` — name of the tool currently executing
- `_tool_count: int` — total tool calls this turn
- `_start_time: float | None` — time.monotonic() when turn started
- `_update_task: asyncio.Task | None` — background 30s refresh loop
- `_bot`, `_chat_id`, `_thread_id` — Telegram targeting

Methods:

`async def start_turn(self) -> None`:
  - Set `_start_time = time.monotonic()`, reset `_tool_count = 0`, `_current_tool = None`
  - Send initial status message: `"⚡ Working..."` with parse_mode="HTML"
  - Store `_message_id` from the sent message
  - Start `_update_task = asyncio.create_task(self._refresh_loop())`

`def track_tool(self, tool_name: str) -> None`:
  - Set `_current_tool = tool_name`
  - Increment `_tool_count += 1`

`async def _refresh_loop(self) -> None`:
  - Loop: `await asyncio.sleep(30)`, then call `_edit_status()`
  - Catch `asyncio.CancelledError` to exit cleanly
  - Catch `TelegramRetryAfter` from aiogram: `await asyncio.sleep(e.retry_after)`, retry once

`async def _edit_status(self) -> None`:
  - If `_message_id` is None, return
  - Build status text using format from CONTEXT.md decisions:
    ```
    ⚡ Working...
    📁 Tool: {_current_tool or "—"}
    ⏱ {elapsed} | {_tool_count} tools
    ```
  - Elapsed = format `time.monotonic() - _start_time` as `Xm Ys`
  - Call `bot.edit_message_text(text=status_text, chat_id=..., message_id=_message_id, message_thread_id=_thread_id, parse_mode="HTML")`
  - Catch `TelegramRetryAfter`: sleep and retry once
  - Catch `TelegramBadRequest` (message not modified): ignore silently

`async def finalize(self, cost_usd: float | None, duration_ms: int, tool_count: int) -> None`:
  - Cancel `_update_task` if running
  - Edit status message to final summary:
    ```
    Done
    Cost: ${cost_usd:.4f} | Duration: {duration_ms/1000:.1f}s | {tool_count} tools
    ```
  - Schedule deletion after 30s: `asyncio.get_running_loop().call_later(30, lambda: asyncio.create_task(self._delete_status()))`

`async def _delete_status(self) -> None`:
  - If `_message_id`, call `bot.delete_message(chat_id, _message_id)`, catch exceptions silently

`async def stop(self) -> None`:
  - Cancel `_update_task` if running, set `_message_id = None`

Create `src/bot/output.py` with two utilities:

**1. `split_message(text: str, max_len: int = 4096) -> list[str]`**

Split long text for Telegram's 4096-char limit, preserving code blocks:

- If `len(text) <= max_len`, return `[text]`
- Strategy: find the last occurrence of "\n```" before the `max_len` boundary. If found, split there (keeping the closing ``` on the first chunk). The second chunk starts a new code block with ``` on the next line.
- Fallback: if no code block boundary found, split at last `\n` before boundary
- Final fallback: hard split at `max_len`
- Apply recursively to remaining text (it may still be >4096)
- Handle edge case: if text starts mid-code-block from a previous split, the leading ``` from the split logic ensures it's valid

**2. `TypingIndicator` class**

Constructor takes: `bot: Bot`, `chat_id: int`, `thread_id: int`.

`async def start(self) -> None`:
  - Create `_task = asyncio.create_task(self._loop())`

`async def _loop(self) -> None`:
  - Loop: call `bot.send_chat_action(chat_id=self._chat_id, action="typing", message_thread_id=self._thread_id)`, then `await asyncio.sleep(4)`
  - Catch `asyncio.CancelledError` to exit cleanly
  - Catch `TelegramRetryAfter`: sleep `e.retry_after`, continue
  - Catch `Exception`: log warning, continue (don't crash typing for transient errors)

`async def stop(self) -> None`:
  - Cancel `_task` if running, await it (catching CancelledError)
