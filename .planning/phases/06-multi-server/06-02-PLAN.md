<objective>
Create the worker process that runs on remote servers, manages ClaudeSDKClient instances locally, and communicates with the central bot via TCP.

Purpose: The worker is the remote execution unit. It reuses the existing SessionRunner but replaces the Bot dependency with a TCP output channel, enabling Claude to run on any server while the Telegram interface stays centralized.

Output: `src/worker/` package with `__main__.py` (entry point), `client.py` (TCP connection + session management), `output_channel.py` (Bot-replacement adapter).
</objective>

1. **src/worker/client.py** — Create `WorkerClient` class:

   **Constructor:** `__init__(self, host: str, port: int, auth_token: str, worker_id: str)`
   - Store connection params
   - `self._sessions: dict[int, SessionRunner] = {}` — local sessions by topic_id
   - `self._output_channel: WorkerOutputChannel | None = None`
   - `self._writer: asyncio.StreamWriter | None = None`
   - `self._permission_futures: dict[str, asyncio.Future] = {}` — pending permission requests

   **`async def run(self)`** — Main reconnect loop (exponential backoff):
   ```
   delay = 1.0
   while True:
       try:
           reader, writer = await asyncio.open_connection(self._host, self._port)
           self._writer = writer
           # Auth handshake
           await send_msg(writer, AuthMsg(token=self._auth_token, worker_id=self._worker_id))
           response = await recv_b2w(reader)
           if not isinstance(response, AuthOkMsg):
               logger.error("Auth rejected: %s", response)
               writer.close(); await writer.wait_closed()
               break  # permanent auth failure — don't retry
           delay = 1.0  # reset on success
           logger.info("Connected to bot IPC as %s", self._worker_id)

           # Update output channel writer for reconnected sessions
           if self._output_channel is None:
               self._output_channel = WorkerOutputChannel(writer, chat_id=0)  # chat_id unused on worker
           else:
               self._output_channel.set_writer(writer)

           # Re-register existing sessions on reconnect
           for topic_id, runner in self._sessions.items():
               if runner.session_id:
                   await send_msg(writer, SessionStartedMsg(topic_id=topic_id, session_id=runner.session_id))

           # Message receive loop
           await self._receive_loop(reader)
       except (ConnectionRefusedError, OSError, asyncio.TimeoutError) as e:
           logger.warning("IPC connect failed: %s — retrying in %.0fs", e, delay)
       finally:
           self._on_disconnected()
       await asyncio.sleep(delay)
       delay = min(delay * 2, 60.0)
   ```

   **`async def _receive_loop(self, reader)`** — Process bot messages:
   ```
   while True:
       msg = await recv_b2w(reader)
       if msg is None: break  # EOF
       if isinstance(msg, StartSessionMsg):
           await self._start_session(msg)
       elif isinstance(msg, StopSessionMsg):
           await self._stop_session(msg.topic_id)
       elif isinstance(msg, UserMessageMsg):
           runner = self._sessions.get(msg.topic_id)
           if runner: await runner.enqueue(msg.text)
       elif isinstance(msg, PermissionResponseMsg):
           self._resolve_permission(msg.request_id, msg.action)
       elif isinstance(msg, SlashCommandMsg):
           runner = self._sessions.get(msg.topic_id)
           if runner: await runner.enqueue(msg.command)
   ```

Since `self._bot` is now `WorkerOutputChannel`, the send_message for permission prompts goes over TCP. BUT the issue is: the keyboard `request_id` is generated worker-side, and the bot needs to map it. Better approach:
