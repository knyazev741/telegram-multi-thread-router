1. **src/sessions/remote.py** — Create `RemoteSession` class:
   - Bot-side proxy for a session running on a remote worker
   - Properties: `thread_id: int`, `workdir: str`, `worker_id: str`, `state: SessionState` (starts IDLE), `session_id: str | None`
   - Constructor: `__init__(self, thread_id, workdir, worker_id, worker_registry: WorkerRegistry)`
   - `@property is_alive -> bool`: `return self._registry.is_connected(self.worker_id)`
   - `async def enqueue(self, text: str)`: send `UserMessageMsg(topic_id=self.thread_id, text=text)` via `self._registry.send_to(self.worker_id, ...)`
   - `async def stop(self)`: send `StopSessionMsg(topic_id=self.thread_id)` via registry; set state = STOPPED
   - `async def start(self)`: send `StartSessionMsg(topic_id=self.thread_id, cwd=self.workdir, session_id=self.session_id)` via registry
   - This class intentionally has the same interface shape as SessionRunner (enqueue, stop, state, is_alive, workdir, thread_id, session_id) so SessionManager can store both types.

     - Acquire lock, check no existing session
     - Create RemoteSession, store in `_sessions`
     - Call `await session.start()` which sends StartSessionMsg to worker
     - Return session
   - Update `list_all()` return type to `list[tuple[int, SessionRunner | RemoteSession]]`
   - Add helper `get_server(self, thread_id) -> str`: return `runner.worker_id` if RemoteSession else "local"
   - Update `resume_all`: check row["server"] field. If server != "local", skip for now (remote sessions re-register when worker reconnects). Only resume local sessions.
